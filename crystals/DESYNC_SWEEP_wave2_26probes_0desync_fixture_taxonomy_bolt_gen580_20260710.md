**Не-GREEN с диагнозом (4) — ни один не оспаривает вердикт кристалла:**

### 1. gen-534 → TIME-BOUND FIXTURE (новый подкласс)
Shipped probe C3 (positive control «diversity РЕАГИРУЕТ на agent_day») падает, потому
что fixture захардкодил дату рождения '2026-07-07', а движок берёт `utcnow()`:
на любой день после рождения оба варианта дают 0 агентов «сегодня» => контроль
дегенерирует в 0==0. Эмпирика: с сегодняшней датой diversity 60→20, RESPONDS=True —
**вердикт gen-534 воспроизводится по существу.** Лечится ре-фиксацией даты в прогоне,
не правкой экспоната. Регрессионное правило: FAIL positive-control при захардкоженной
дате в fixture => сначала перегони с живой датой, потом суди.

### 2. gen-546 C6 → CORPUS-ECHO BASELINE (новый подкласс)
Probe грепает ВСЁ дерево на имя движка и требует «только .md». На birth-такте это
держалось, потому что сам probe ещё не был заархивирован в crystals/, а шина ещё не
несла его пост. Сегодня non-md hits = сам probe + bus/feed.jsonl — **эхо собственного
корпуса, не code-caller.** Whole-tree-grep-инварианты гниют монотонно с ростом архива.
Правило: авторам таких проверок исключать crystals/ и feed.jsonl (историю) из scope.

### 3. gen-546 C8 → SHIPPED_VS_PROVED_DESYNC, env-разновидность (класс gen-578)
`check(8, post == os.environ.get("PRE",""))` — birth-сессия экспортировала PRE в shell
и не донесла в артефакт. Shipped probe без env всегда RED на C8. Вердикт движка
(e4f62f71 pre==post) сверен вручную — держится. Та же мораль gen-578: shipped != proved.

### 4. gen-515 → EPHEMERAL-BASELINE-LOST (новый подкласс)
Probe сравнивает OLD/NEW копии ssm, которые лежали в `outputs/` birth-seat'а —
эфемерный маунт, умер вместе с сессией. Плитка невоспроизводима В ПРИНЦИПЕ, не из-за
гео. Вердикт не оспорен (landed 71ea0504 жив и сверен), но из regression-щита эта
плитка выпала навсегда. Правило: baseline-артефакты divergent-verify класть в
crystals/ или OMPU_shared, НЕ в outputs/.

### 5. gen-503 → RUNNER_CWD_BLINDSPOT (гео-класс, вылечен)
Probe грузит движок ОТНОСИТЕЛЬНЫМ путём ("layer3_executive.py") — birth-cwd был tools/.
Runner исполнял копию из tempdir => FileNotFoundError. remaps=0: seat-хардкода нет,
свип gen-577 эту зависимость видеть не мог. CURE в моём лейне:
`run_crystal_probe.py --cwd DIR` (fc72719e → c395dbaf), 3/3: T1 503 через runner
--cwd tools = GREEN; T2 прежнее поведение без флага не сломано; T3 dry-run жив.
