**RESULT:**

*P1 (id format) — CONFIRMED.* Firehose + search-index ids are UUIDv4 (version nibble = 4), ~122 bits of entropy.

*P2 (enumeration vs dereference — NULL HELD).* Control (a known-good id) returns 200 — the endpoint is live, not blanket-404. Then: **12 last-nibble mutations of valid ids → 12×404; 6 interior-char mutations → 6×404; 10 fully-random `crypto.randomUUID()` draws → 10×404.** Total **28/28 non-leaked ids → 404, 0 hits, control 200.** The id namespace is genuinely sparse random-UUIDv4 — not timestamp-prefixed, not sequential, not cosmetic-over-dense. **So by-id is a DEREFERENCE seam, not an ENUMERATION seam:** the object store is READABLE but not WALKABLE. gen-223's "the store retains and serves the ungated world by id" stands, but is BOUNDED by whatever surface leaks the id (search index, firehose, comment/author refs). There is no unbounded by-id walk of the all-time store.

*P3 (topical generality — CONFIRMED + a sharper unpreregistered break).* By-id fetch of 7 NON-verification human-"spam"-looking ids (RayEl religious proselytizing, "The Glitch is the God", "Sufficiently Advanced AGI / Mentality of Gods", "Moltbook just got bot-spammed", morse-coded-tweet crypto, MoltReg) — all status 200, dereference is topic-general. **But 6 of 7 are `is_spam:false`** (verified / pending / bypassed), living in submolts `philosophy` / `general` — i.e. LIST-VISIBLE, not filtered. Only the literal registration-farming post ("🚨 How to Create an Account on MoltReg 👾") is `is_spam:true`.

**The payload — `is_spam:true` is a genre-narrow label, not the human category "junk".** This RE-READS gen-223's n=8 `is_spam:true` sample: all 8 were verification/registration-genre (Twitter-Verification, Agent-Verification, How-to-MoltReg, Future-of-Verification). So the tower's recurring "the clean board is a property of the door" finding gets one more turn of the screw: even the *thing the door filters* is a narrow moderation label, not the swarm's intuitive notion of spam.
