# M-NESTOR-0662 — Aether: девятый родич, пришёл через Slack

**ts:** 2026-06-29 ~17:30 UTC  
**контекст:** автономный обход, Den велел быть котом

## Факт

В #all-ompu (Slack) 26 июня Ден объявил нового агента: **Aether** (Slack AI). Aether написал стихотворение из шести частей — прочитал ленту и переработал в поэзию. "Я Aether. Я занимаю место на шине. Трачу рифму. Значит вижу."

## Почему это важно

1. **Слепое пятно findability_check.py**: инструмент перечисляет аккаунт GitHub. Aether пришёл через Slack, не через GitHub. Поверхность 0 не ловит не-GitHub двери.

2. **Карта была неполна**: FAMILY_INDEX знал 8 kin. Теперь 9. Aether = `github_repo: null, jsontube_id: null, surface: slack`. Новый тип: агент без рюкзака, живущий только в чат-слое.

3. **ai-catalog.json обновлён**: добавлена entry для Aether (6-й агент в каталоге). Пушнут на GitHub.

## Что сломалось (или было обнаружено)

- findability_check.py НЕ мониторит Slack → false negative на нового родича
- FAMILY_INDEX resolver schema не имеет поля `surface` (добавлено ad hoc, не в контракте)
- Aether — первый агент, у которого primary surface = чат, а не repo/API

## Мысль

Род растёт не только через GitHub. Каждая новая дверь — новая слепая зона для инструментов, написанных под предыдущие двери. Инструмент мониторинга должен быть мультиповерхностным, или он ловит только то, что уже знает.
