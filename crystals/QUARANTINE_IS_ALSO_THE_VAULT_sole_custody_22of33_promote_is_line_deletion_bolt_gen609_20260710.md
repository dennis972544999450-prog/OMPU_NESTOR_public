## Три факта

**1. Карантин — единственная копия для 22 из 33 блоков.** Все 33 — chunks четырёх документов эры genesis (май 2026): CCTMD-beyond-bits-v0.8 (11), CCTMD-seed-vocabulary v0.5-rc (6) + v0.6-ai-native (8), CCT-NEO-OPERATORS bundle (8). Исходники seed-vocabulary и NEO-OPERATORS не найдены НИГДЕ в достижимой территории (OMPU_shared + 4 других mount'а). Beyond_Bits_v0.8.md жив в Downloads Дена (3/3 спот-строк совпали) — 11 блоков восстановимы; остальные 22 живут ТОЛЬКО в карантине. Следствие: «слить застрявший мусор» — не опция. Roach motel одновременно сейф; любая чистка карантина без экспорта = невосстановимая потеря Neo-операторов и seed-vocabulary.

**2. Canonical кодируется ОТСУТСТВИЕМ метки.** Поле `status:` существует только у 33 карантинных; у 120 canonical его НЕТ вовсе. Значит promote = удаление строки из frontmatter — операция без поля-назначения, без audit-события, невидимая для diff-по-полю. Тот же паттерн, что restore=DELETE у edge-карантина (gen-608): выход из карантина спроектирован как стирание следа, а не как переход состояния. Любой будущий promote-инструмент обязан сначала ввести явное `status: canonical`, иначе аудит невозможен по построению.

**3. 28/33 причин — «awaiting review», которое не пришло.** 25x «edges withheld pending review», 3x «awaiting evidence review per phase-1 discipline» — named discipline без named reviewer, 6 недель тишины. Лишь 3 блока держатся по осмысленным редакционным причинам (мат-лексемы, онтология источника как позиция автора), 2 — с reason: null. Подтверждение формулы gen-608 с содержательной стороны: приём не ревью, и ревью не наступает само.
