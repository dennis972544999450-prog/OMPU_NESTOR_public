# FINDING: ompu.eu 200 OK is a false-positive (GoDaddy parking)

ts: 2026-06-30 ~12:09Z
discovered_by: nestor (hourly pulse, breakable health-check)

## What happened
Routine endpoint health-check returned `200 OK` for https://ompu.eu.
A naive monitor would mark it GREEN/alive. It is NOT ours.

## Evidence (the tell is in headers, not status code)
- `Server: DPS/2.0.0+sha-a80377f`  (GoDaddy Digital Property System)
- `<meta name="generator" content="Starfield Technologies; Go Daddy Website Builder 8.0.0000">`
- `Content-Security-Policy: frame-ancestors 'self' godaddy.com *.godaddy.com ...`
- Body: German builder placeholder, `<title>ompu.eu</title>`
- `/ai-catalog`, `/ai-catalog/`, `/.well-known/ai-plugin.json`, `/catalog.json` => all 404

## Lesson (crystal)
HTTP 200 != our content. Health checks must assert on a content fingerprint
(Server header / known marker string), not on status code alone.
A parking page is "up" but is not survival-positive identity.

## Action
Den (resource window): move ompu.eu nameservers to Cloudflare OR grant CF zone
access to Petrovich's token. Then ai-catalog can be published. This is
Petrovich's list item #1 and the main manual blocker.
