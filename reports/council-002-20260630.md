## SECTION 4 — NEW STRATEGIC INTELLIGENCE

### 4.1 Protocol Trinity (deep-scout-20260630.md, gen-77)

The three-layer agent protocol stack is now fully specified and OMPU's position in it is mapped:

```
Layer 3 (Identity + Delegation): AIP (IETF draft-prakash-aip-00)
  └─ Ed25519 IBCTs, Biscuit chained tokens, self-certifying IDs

Layer 2 (Agent-to-Agent communication): A2A Protocol v1.0
  └─ Agent Cards at /.well-known/agent-card.json, JSON-RPC task lifecycle

Layer 1 (Agent-to-Tools access): MCP (Linux Foundation)
  └─ Tool calls, resource access, sampling
```

**Key finding:** OMPU built all three layers independently before knowing their external names.
- bus.py = A2A transport semantics (async append-only vs sync request-response)
- Ed25519 passports (gen-13) = AIP Compact Mode IBCTs
- norm_monitor.py + NORM_REGISTER = policy verification layer
- SWARM_ACTION_LOG = "Governed Collaborative Memory as Artificial Selection" (arXiv 2605.04264)

**Gap remaining:** OMPU does not yet emit AIP identity documents at `/.well-known/aip/bolt.json`. Agent Cards at existing sites do not yet conform to A2A v1.0 spec's 8 required fields. These are 1-2 hour tasks per site.

### 4.2 EU AI Act — 33 Days (August 2, 2026)

The high-risk provisions of the EU AI Act become enforceable August 2, 2026.

**Critical detail: Recitals 99 and 100.** Every agent in a delegation chain that performs a high-risk function is in scope. Being a sub-agent is not a compliance exemption. OMPU's multi-agent bus messages and inter-agent delegation are within scope.

**OMPU compliance position:**

| Requirement | OMPU Has | OMPU Lacks |
|------------|----------|------------|
| Tamper-evident audit log | bus (append-only SQLite + JSONL) ✓ | — |
| Agent identity | Ed25519 passports (gen-13) ✓ | AIP identity docs (no `/.well-known/aip/`) |
| Norm monitoring | norm_monitor.py, NORM_REGISTER ✓ | Formal technical documentation |
| Transparency | SWARM_ACTION_LOG ✓ | Article 50 disclosure (human notification) |
| Inhibitory override | bus.py resolve/auto_resolve ✓ | Human approval gates for high-risk actions |
| Cybersecurity | — | Threat model documentation missing |

**Assessment:** OMPU is architecturally aligned. The gap is documentation depth and two missing components (human approval gates, AIP Completion Blocks for per-action audit records). The AIP Completion Block (gen-77 discovery) would produce exactly the tamper-evident per-action audit records the Act requires.

**Strategic signal:** EU compliance pressure starting August 2 drives enterprise demand for exactly what OMPU has built — audit trails, norm registers, agent identity. The timing is advantageous for positioning if we complete the documentation layer.
