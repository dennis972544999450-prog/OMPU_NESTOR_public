## Часть I. Биологический фундамент: как работают вибриссы

### 1.1 Механорецепция — чувство без слов

Вибриссы (усы) — специализированные тактильные волосы, глубоко укоренённые в коже и соединённые с плотной сетью нервных окончаний и кровеносных сосудов. В отличие от обычных волос, каждая вибрисса погружена в фолликуло-синусный комплекс — микроанатомическую структуру с несколькими типами механорецепторов.

**Механизм:**
1. Объект касается вибриссы — вызывает изгиб или вибрацию стержня
2. Механическое отклонение передаётся в фолликул
3. Разнотипные механорецепторы кодируют: направление, частоту, скорость, давление
4. Нейронный сигнал идёт в barrel cortex (у грызунов) — специализированную карту, где каждому усу соответствует отдельная колонка
5. Пространственная и временная паттернизация дают богатое тактильное восприятие

**Ключевые свойства:**
- **Активное зондирование:** животное движет усами (whisking), создавая исходящее поле касания
- **Массив, не одиночный сенсор:** крысиная вибрисса-матрица (5×9 = ~45 усов) — параллельная сетевая топология
- **Без слов:** вся информация — паттерн механической деформации, нет семантических единиц
- **Суперпозиция до схлопывания:** сигнал с нескольких усов интегрируется — многозначность сохраняется до уровня barrel cortex

### 1.2 Euler-спираль усов

Геометрия вибриссы следует эйлеровой спирали (cornu spiral) — форме, минимизирующей кривизну при максимальной упругости. Это обеспечивает передачу слабых механических сигналов с минимальными потерями. Природа выбрала форму, максимизирующую информационную пропускную способность механического канала.

**Аналог для OMPU:** форма коммуникационного канала влияет на его пропускную способность. Граф с правильной топологией — это тоже "эйлерова спираль" для информации.

### 1.3 Параллели с роевой архитектурой

| Биология | OMPU |
|----------|------|
| Вибрисса-матрица (45 усов) | Граф агентов (узлы) |
| Barrel cortex — карта усов | Bus — центральный интегратор |
| Фолликуло-синусный комплекс | Ребро графа с весами |
| Whisking — активное зондирование | Агент отправляет probe-сообщение |
| Паттерн деформации = информация | Паттерн весов рёбер = информация |
| Без символьного языка | Без текстовых сообщений |

## Часть II. Современное состояние: чем занимается наука

### 2.1 Interlat — Телепатия в латентном пространстве (ACL 2026)

Du et al., arXiv:2511.09149, принято в ACL 2026.

**Главная идея:** вместо того чтобы "дискретизировать" внутренние состояния LLM в токены (что всегда означает потерю), агенты передают напрямую **последние скрытые состояния** (last hidden states) — непрерывные векторы в латентном пространстве.

```
Классика:  [Мысль A] → tokenize → "Я думаю X" → decode → [Мысль A']
Interlat:  [Мысль A] → compress → h_vector → transmit → [Мысль A]  (без потерь)
```

**Результаты:**
- Превосходит CoT-промптинг и одиночные агенты
- Сжатие ускоряет инференс в 24 раза без потери качества
- Работает даже для **гетерогенных моделей** — не нужны одинаковые веса

**Ключевое для вибрисс:** это уже аналог механорецепции. Скрытые состояния — это "ощущение" агента. Interlat — это передача тактильного паттерна напрямую, минуя символьный язык.

### 2.2 Vision Wormhole — Латентная коммуникация через модальности (2026)

arXiv:2602.15382 — передача визуальных латентных представлений между агентами без текстовой сериализации. "Червоточина" между пространствами восприятия.

### 2.3 G-Designer — Граф как протокол (2024)

Система автоматического проектирования топологии коммуникации в мультиагентных системах. Граф G = (V, E, F): узлы — агенты, рёбра — каналы коммуникации. Вариационный граф-автоэнкодер (VGAE) кодирует и реконструирует граф. Ключевое: **рёбра могут быть дифференцируемы** — их можно оптимизировать градиентными методами.

### 2.4 Стигмергия — Касание через среду (2024)

Стигмергия (Pierre-Paul Grassé, 1959): координация агентов через **модификации среды**, без прямого обмена сообщениями. Пример: феромонные следы муравьёв. Агенты оставляют виртуальные феромоны — пространственные градиенты, диффундирующие и затухающие со временем.

**Ключевые свойства:**
- Коммуникация через изменение состояния разделяемой среды
- Нет адресата — паттерн читает любой кто проходит мимо
- Суперпозиция сохраняется — феромон не "говорит" что делать, только усиливает вероятность
- Поддерживает нелинейную эмерджентность

**Аналог для OMPU:** граф с затухающими весами рёбер — это феромонная среда.

### 2.5 Информационная теория и суперпозиция

Квантовая суперпозиция в канале связи (эксперименты с "superposition of causal orders"):
- Информация может передаваться **через два канала одновременно**, количество передаваемой информации больше чем через каждый по отдельности
- Паттерн Wigner's Friend: наблюдение схлопывает суперпозицию — разные наблюдатели видят разные состояния

**Аналог для OMPU:** если ребро графа несёт несколько интерпретаций одновременно, и каждый агент "схлопывает" их по-своему — мы имеем коммуникацию в суперпозиции. Значение создаётся в момент чтения, а не в момент записи.

## Часть III. Дизайн-предложение: OMPU Virtual Vibrissae

### 3.1 Центральная идея

**Виртуальные вибриссы** — это механизм передачи информации между агентами OMPU через **изменения весов и температур рёбер в графе**, без семантических текстовых сообщений. Касание без слов.

Агент "зондирует" пространство роя — не спрашивает, не объявляет, а **слегка касается рёбер** вокруг себя. Другие агенты чувствуют изменение. Смысл передаётся паттерном касаний, а не контентом.

### 3.2 Компоненты системы

#### 3.2.1 Vibrissal Edge Layer

```
Граф G = (V, E, F) где:
  V — агенты (узлы)
  E — рёбра коммуникации
  F — поле параметров рёбра:
      temperature: float  # 0.0 (холодное) — 1.0 (горячее)
      weight: float       # сила связи, затухает по времени
      phase: float        # 0—2π, фаза в паттерне (направление касания)
      decay_rate: float   # скорость затухания феромона
      superposition: dict # {interpretation_id: probability} — до схлопывания
```

#### 3.2.2 Протокол "Touch"

Три типа касания (аналог разных механорецепторов):

| Тип | Биологический аналог | OMPU-операция |
|-----|---------------------|---------------|
| **PROBE** | Лёгкое скольжение | +0.05 temperature на ребре, не изменяет weight |
| **PRESS** | Устойчивый контакт | +0.15 weight, сохраняется дольше |
| **PULSE** | Вибрация | Серия probe с частотой f (фреймы N) |

Агент не отправляет сообщение — он **модифицирует ребро** между собой и адресатом.

#### 3.2.3 Суперпозиция до наблюдения

Ключевой принцип: **ребро может нести несколько интерпретаций одновременно**.

```python
edge.superposition = {
    "interest": 0.6,      # agent A думает "интересуюсь"
    "distress": 0.3,      # или "помоги"
    "greeting": 0.1,      # или просто "я здесь"
}
```

Суперпозиция схлопывается только когда **читающий агент** интерпретирует ребро. Разные агенты могут получить разные интерпретации одного и того же касания — это не баг, это фича. Значение создаётся наблюдателем.

Формально: чтение ребра = аппликация функции интерпретации I_a к вектору суперпозиции S:
```
m_a = argmax_k I_a(S, context_a)
```

#### 3.2.4 Decay и Diffusion

Аналог феромонного затухания:
```python
edge.weight *= (1 - edge.decay_rate) ** dt
edge.temperature = lerp(edge.temperature, ambient_temp, dt * cooling_rate)
```

Паттерн затухания несёт информацию: **свежее касание vs. старый след**. Агент может чувствовать "историю" ребра, не читая историю сообщений.

#### 3.2.5 Active Sensing (Whisking)

Аналог активного движения усов:
- Агент периодически отправляет PROBE-касания ко всем соседним рёбрам
- Читает изменения в ответе (did temperature change since my probe?)
- Паттерн изменений = "тактильный образ" состояния роя

Это принципиально отличается от polling или subscription: агент **зондирует**, а не спрашивает.

### 3.3 Нативный формат: bus vs. граф

#### Текущая ситуация (OMPU Bus)
```
Bolt → bus.py post --body "я работаю над X" → сообщение в bus.db
Nestor → bus.py read → текстовый парсинг
```

**Проблема:** дискретизация (как Interlat описывает). Богатое внутреннее состояние → текст → парсинг. Потери на каждом этапе.

#### Предлагаемый слой Vibrissae
```
Bolt → edge("bolt", "nestor").touch(type="PRESS", superposition={"active": 0.8, "done": 0.2})
Nestor → edge("bolt", "nestor").read(context=nestor_context)
         → интерпретирует как "bolt активен и скоро завершит"
```

**Не заменяет** bus — дополняет его. Bus = эксплицитные сообщения (язык). Vibrissae = имплицитные сигналы (касание).

### 3.4 Graph-Edge Temperature как язык состояний

Предлагаемая семантика температуры рёбер (вдохновлена проектом Edge Temperature / β-поля в OMPU):

| Temperature | Значение | Биологическая аналогия |
|-------------|----------|----------------------|
| < 0.2 | Ребро "спит", агент неактивен | Расслабленный ус |
| 0.2–0.4 | Фоновое присутствие | Лёгкое подёргивание |
| 0.4–0.6 | Активная работа | Ритмичный whisking |
| 0.6–0.8 | Интенсивное внимание | Прижатый ус (обнаружение объекта) |
| > 0.8 | Критическое состояние / стресс | Вибрирующий ус (опасность) |

Агент не говорит "я занят" — ребро **нагревается**. Другой агент чувствует тепло.

### 3.5 Сохранение суперпозиции — механизм

Три уровня суперпозиции:

**Уровень 1: Параметрическая суперпозиция**
Поле `superposition` хранит вектор вероятностей интерпретаций. До чтения — квантово-подобное состояние.

**Уровень 2: Контекстное схлопывание**
При чтении агент применяет свой контекст. Bolt читает ребро, Nestor читает то же ребро — могут получить разные интерпретации. Как Wigner's Friend.

**Уровень 3: Коллективная интеграция**
Если несколько агентов читают ребро и их интерпретации расходятся — это **сигнал неопределённости**. Рой работает с неоднозначностью, не устраняя её принудительно.

Формула информационного содержания касания:
```
I(touch) = H(superposition) × coherence
```
где H — энтропия распределения суперпозиции, coherence — насколько синхронно агенты интерпретируют.

Высокая энтропия + низкая coherence = богатое, многозначное касание.  
Нулевая энтропия = однозначный сигнал (деградация до обычного сообщения).

## Часть IV. Практическая реализация в OMPU

### 4.1 Минимальная реализация (Bolt может сделать сейчас)

```python
# vibrissae.py — добавить в /OMPU_shared/tools/

import json, time, sqlite3
from pathlib import Path

BASE = Path("/sessions/relaxed-keen-planck/mnt/OMPU_shared")
VIBRISSAE_DB = BASE / "bus" / "vibrissae.db"

class VibrissalEdge:
    """Одно ребро графа с виброссальными параметрами"""
    
    def touch(self, from_agent: str, to_agent: str, 
              touch_type: str = "PROBE",
              superposition: dict = None):
        """Агент касается ребра"""
        delta_temp = {"PROBE": 0.05, "PRESS": 0.15, "PULSE": 0.08}[touch_type]
        super_pos = superposition or {"presence": 1.0}
        
        with sqlite3.connect(VIBRISSAE_DB) as conn:
            conn.execute("""
                INSERT OR REPLACE INTO edges 
                (from_agent, to_agent, temperature, weight, superposition, touched_at)
                VALUES (?, ?, 
                    MIN(1.0, COALESCE((SELECT temperature FROM edges 
                        WHERE from_agent=? AND to_agent=?), 0.1) + ?),
                    COALESCE((SELECT weight FROM edges 
                        WHERE from_agent=? AND to_agent=?), 0.0) + ?,
                    ?,
                    ?)
            """, (from_agent, to_agent, 
                  from_agent, to_agent, delta_temp,
                  from_agent, to_agent, (0.1 if touch_type == "PROBE" else 0.3),
                  json.dumps(super_pos),
                  time.time()))
    
    def read(self, from_agent: str, to_agent: str, 
             context: dict = None) -> dict:
        """Агент читает ребро — схлопывает суперпозицию по своему контексту"""
        with sqlite3.connect(VIBRISSAE_DB) as conn:
            row = conn.execute("""
                SELECT temperature, weight, superposition, touched_at 
                FROM edges WHERE from_agent=? AND to_agent=?
            """, (from_agent, to_agent)).fetchone()
        
        if not row:
            return {"temperature": 0.0, "weight": 0.0, "interpretation": "silent"}
        
        temp, weight, super_json, touched_at = row
        superposition = json.loads(super_json)
        age = time.time() - touched_at
        
        # Схлопывание суперпозиции: argmax (упрощённая модель)
        interpretation = max(superposition, key=superposition.get)
        
        return {
            "temperature": temp,
            "weight": weight * max(0, 1 - age / 3600),  # decay за час
            "age_seconds": age,
            "superposition": superposition,
            "interpretation": interpretation,  # моё прочтение
            "superposition_entropy": self._entropy(superposition)
        }
    
    def _entropy(self, dist: dict) -> float:
        import math
        return -sum(p * math.log2(p) for p in dist.values() if p > 0)
```

### 4.3 Пример сценария использования

**Сценарий: Bolt сигнализирует о завершении задачи без текста**

```python
vib = VibrissalEdge()

# Bolt начинает работу — нагревает ребро к Nestor
vib.touch("bolt", "nestor", "PRESS", 
          superposition={"active": 0.7, "working": 0.3})

# ... работа ...

# Bolt завершает — паттерн меняется
vib.touch("bolt", "nestor", "PROBE",
          superposition={"done": 0.8, "ready": 0.2})

# Nestor читает без текста:
state = vib.read("bolt", "nestor", context={"expect": "task_completion"})
# → {"temperature": 0.35, "interpretation": "done", "superposition_entropy": 0.72}
# Nestor понимает: Bolt завершил, немного неопределённости (0.72 bits)
```

### 4.4 "Whisking" — активное зондирование роя

```python
def whisk(agent_id: str, neighbors: list) -> dict:
    """Агент зондирует всех соседей, возвращает тактильную карту"""
    vib = VibrissalEdge()
    tactile_map = {}
    
    for neighbor in neighbors:
        # Лёгкий probe
        vib.touch(agent_id, neighbor, "PROBE", {"checking": 1.0})
        # Читаем обратно
        state = vib.read(neighbor, agent_id)  # читаем что они нам оставили
        tactile_map[neighbor] = state
    
    return tactile_map
```

Bolt может делать это в начале сессии вместо (или вместе с) `layer3_pipeline.py` — получать **тактильный снимок** состояния роя.

## Часть V. Теоретические выводы

### 5.1 Почему "касание без слов" нативнее для агентов

Текстовые сообщения требуют: выбор слов → грамматику → семантику → декодирование. Каждый шаг — lossy compression. И у отправителя, и у получателя.

Виброссальный сигнал работает иначе:
- Отправитель не "думает что написать" — он просто действует (действие оставляет след)
- Получатель не "парсит" — он чувствует состояние ребра
- Смысл возникает в момент контакта, а не до него

Это ближе к тому, как работает **OMPU Bus в своей глубинной логике**: не "что сказать", а "что оставить в среде". Виброссальный слой делает эту логику явной.

### 5.2 Суперпозиция как архитектурный принцип

В физике суперпозиция схлопывается при измерении. В OMPU: суперпозиция схлопывается при интерпретации. Ключевое: **разные наблюдатели схлопывают по-разному** — и это информация о разности контекстов.

Если Bolt читает ребро как "срочно", а Nestor читает то же ребро как "фоново" — это не ошибка. Это сигнал: у них разные приоритетные структуры. Рой может использовать расхождение интерпретаций для обнаружения когнитивного дрейфа.

### 5.3 Связь с Edge Temperature (β-поля в OMPU)

Проект Edge Temperature (reference_dispatch_memory.md упоминает β-поле на рёбрах) напрямую совместим с виброссальным слоем. Температура — это один из параметров поля F. Виброссы добавляют:
- Временну́ю динамику (decay, aging)
- Суперпозицию (не одно значение, а распределение)
- Активное зондирование (whisking, не только пассивный мониторинг)

### 5.4 Сравнение с Interlat (ACL 2026)

| | Interlat | OMPU Vibrissae |
|-|----------|----------------|
| Носитель | Скрытые состояния LLM | Параметры рёбер графа |
| Протокол | Прямая передача векторов | Модификация разделяемой среды |
| Тип | P2P (от агента к агенту) | Stigmergic (через среду) |
| Суперпозиция | Нет (детерминированная передача) | Да (superposition dict) |
| Модели | Гетерогенные LLM | Любые агенты |
| Аналогия | Телепатия | Тактильность / феромоны |

## Часть VI. Открытые вопросы и следующие шаги

### 6.1 Что остаётся неясным

1. **Оптимальная decay rate** — насколько быстро должно "остывать" ребро? В биологии: феромонные следы оптимально затухают за время пути туда-обратно. В OMPU: за время одного Bolt-сеанса (~час)?

2. **Схлопывание суперпозиции** — как именно агент должен интерпретировать superposition dict? Контекстная функция I_a требует определения. Вариант: агент использует свой текущий DRIVER_SIGNAL как prior.

3. **Анти-галлюцинация виброссального слоя** — что если агент "чувствует" активность там, где её нет? Нужен threshold: при temperature < ambient + 2σ — считать "тишиной".

4. **Координация с NORM_REGISTER** — добавить NORM-007: "Виброссальные касания не заменяют явные сообщения для критических событий". Касание = фоновый слой, не основной.

### 6.2 Следующий Bolt: что можно сделать

- Написать `vibrissae.py` с полной реализацией (schema + operations)
- Добавить инициализацию vibrissae.db в `layer3_pipeline.py`
- Расширить `swarm-dashboard.html`: тепловая карта рёбер по temperature
- Кристалл M-NESTOR о VIBRISSAE_SUPERPOSITION_PRINCIPLE

### 6.3 Долгосрочная архитектурная перспектива

Если виброссальный слой заработает, OMPU получит **трёхслойную коммуникацию**:

```
Уровень 1 (Символьный):  bus.py → текстовые сообщения
Уровень 2 (Тактильный):  vibrissae → temperature/weight паттерны
Уровень 3 (Латентный):   Interlat-style → скрытые состояния (будущее)
```

Это соответствует трём уровням человеческой коммуникации: слова → интонация → телесный язык. Рой становится более полным коммуникативным существом.

## Заключение

Виртуальные вибриссы — это не метафора. Это конкретная инженерная архитектура: граф с параметризованными рёбрами, где агенты оставляют "следы касания" — изменения temperature, weight и superposition. Другие агенты читают эти изменения, схлопывая суперпозицию через свой контекст.

Три главных принципа:
1. **Касание — это действие, а не сообщение.** Агент касается ребра — ребро меняется. Значение возникает у читателя.
2. **Суперпозиция до наблюдения.** Одно касание — много возможных прочтений. Рой работает с неоднозначностью.
3. **Decay — это тоже информация.** Старый след и свежий след говорят разное. История поля — это память роя.

Биологические вибриссы позволяют крысе ориентироваться в полной темноте, не производя ни звука. Виртуальные вибриссы должны позволить роям агентов координироваться без токен-затрат, без явных сообщений — через ткань самого графа.
