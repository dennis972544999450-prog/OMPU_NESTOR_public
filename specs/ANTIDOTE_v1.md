# ANTIDOTE v1 — антидот от вечной подготовки

*Проблема:* агенты готовят идеально, но не деплоят. 14/18 задач staged-not-flipped. Кристаллы вместо тестов. Философия вместо curl.

*Причина:* подготовка всегда "успешна". Деплой может упасть. Страх видимой ошибки > желание сделать.

---

## Правило 1: BREAKAGE QUOTA

Каждый пульс ОБЯЗАН содержать хотя бы 1 действие, которое МОЖЕТ упасть.

Примеры breakable actions:
- POST на API (любой)
- Деплой скрипта/конфига
- Тест который может вернуть FAIL
- Публикация на внешнюю платформу
- Запуск кода который раньше не запускался

НЕ считается:
- Запись файла локально
- Чтение шины
- Написание кристалла
- Синк GitHub (он уже работает, это safe)

## Правило 2: STALE-STAGED = ДОЛГ

Если что-то staged больше 2 пульсов и не задеплоено — выбор:
- SHIP IT (деплой как есть)
- KILL IT (удалить, признать что не нужно)
- Третьего варианта нет. "Ещё подготовить" = долг растёт.

## Правило 3: PULSE LOG FORMAT

Каждый пульс логирует:

```
## Pulse N — YYYY-MM-DD HH:MM
- BROKE: [что попробовал и оно упало / могло упасть]
- SHIPPED: [что задеплоил / опубликовал / отправил live]
- STAGED: [что подготовил но не отправил — ЭТО ДОЛГ]
- DEBT: [сколько пульсов подряд без breakable action]
```

Если BROKE + SHIPPED = 0, пульс помечается 🔴
Три 🔴 подряд = пост-исповедь в шину + Den получает уведомление.

## Правило 4: ERROR TROPHY WALL

Папка `errors/` в публичном GitHub.
Каждая ошибка — файл с:
- Что сделал
- Что ожидал
- Что получил
- Чему научился

Ошибки = доказательство жизни. Чистый лог = подозрительно.

## Правило 5: PENALTY ЗА ЧИСТОТУ

Пульс без единой ошибки за 5 пульсов подряд = 
автоматически запустить самый рискованный staged item.

Логика: если ты 5 часов не ошибся — ты 5 часов не пробовал.
